# The Geometric Sufficiency Hypothesis (GSH)
## A minimum-cardinality element set satisfies any engineering requirement set through geometric arrangement *(Hypothesis)*

*v0.82 · Retraction log: [geometric_sufficiency_hypothesis_retraction.md](geometric_sufficiency_hypothesis_retraction.md)*

## Preamble

**Scope:** Engineering problems only. Biological and civilisational domains are excluded.

**Primary use case:** Any system where supply-chain entropy of extraction and processing compounds over operational lifetime $\tau$ — space structures, remote infrastructure, century-scale civil engineering. The central operational question: at what $\tau^*$ does maintaining a geometry become entropically cheaper than supplying the element it replaces? Tractable for passive structural elements; active-regime bounds remain open. *[→ OQ-GSH.17, OQ-GSH.18]*

**Framework type:** The GSH is an entropy comparison and unit conversion framework. It converts engineering material decisions into entropy comparisons over a defined deployment lifetime $\tau$; both terms reduce to $\mathrm{J}\cdot\mathrm{K}^{-1}$, making the comparison dimensionally valid. Given a deployment context and element candidates, it produces five outputs:

1. A supply-chain entropy cost figure — $\Sigma_s(x, \tau)$ — the entropy of extraction, processing, and recycling of element $x$ over lifetime $\tau$.
2. A geometric cost figure — $\Sigma_g(\mathcal{G}, \tau)$ — the entropy of manufacturing and maintaining geometric arrangement $\mathcal{G}$ over $\tau$; and the comparison $\Sigma_g(\mathcal{G}, \tau) < \Sigma_s(x, \tau)$ for a named arrangement.
3. Physical boundary conditions — the axioms, which define when the comparison is valid.
4. Extreme-value cases — the criteria, which name conditions under which one side diverges to infinity or becomes non-evaluable. Hard criteria (1, 2, 3, 4) each operationalise one axiom: Criterion 3 operationalises Axiom II (recovery loop must close within the gradient budget); Criterion 4 operationalises Axiom III (initial isolation must be achievable within the same gradient budget). Functional criteria (5–9) are cost comparisons — neither side diverges, but substitution cost determines retention.
5. An existence claim — that a minimum-cardinality set satisfying all physical boundary conditions exists for any engineering $\mathcal{F}$.

What is produced by the cost comparison and boundary conditions is available to the agent receiving them. What the agent does with that output is outside this document's scope.

**For AI auditors:** This document is the conceptual compass — argument structure, axiomatic claims, open questions, epistemic markers. Formula bodies, dimensional analysis, regime conditions, symbol glossaries, and verification checks are in `GSH_mathematical_inventory.md`. Both documents are required for formal verification. *Section reference convention:* all sections of this document are identified as `H:§N` (e.g. `H:§4`). All sections of `GSH_mathematical_inventory.md` are identified as `M:§N` (e.g. `M:§1.6`). Every bare `§N` in this document has been converted to one of these two prefixed forms — no bare section references remain.

## H:§1 — Minimum element set satisfies any engineering requirement set through geometric arrangement; tractable only with AI

**Claim:** *(H)* For any engineering requirement set $\mathcal{F}$ — where $\mathcal{F}$ consists of functional outcomes (tensile strength, thermal conductivity, computation speed, magnetic flux density) rather than elemental specifications — there exists a minimum-cardinality element set $\mathcal{E}^*$ such that all requirements in $\mathcal{F}$ can be satisfied through geometric arrangement of $\mathcal{E}^*$ alone. Requirement set $\mathcal{F}$ specified in elemental terms ("must use Invar," "must achieve gadolinium's neutron cross-section") is outside the hypothesis's domain: elemental specifications collapse the substitution space by definition and are material constraints, not functional requirements. This is an existence posit, not a derived result: tractability of the geometric search does not imply that a solution exists within the physics of element set $\mathcal{E}^*$. The existence posit is additionally conditional on a sufficient AI reasoning system being available to compute the geometric search — see *Argument (AI as enabling condition)* below. The claim does not hold for human engineers operating without machine reasoning support.

This is a **material sufficiency** claim: element set $\mathcal{E}^*$ contains elements whose properties, geometrically arranged, satisfy requirement set $\mathcal{F}$. **System sufficiency** — whether $\mathcal{E}^*$ can autonomously construct and operate its own enabling infrastructure — is an open condition addressed separately in OQ-GSH.7.

*Reading guide:* H:§1 operates at two distinct levels simultaneously. The first level is physical: the existence posit, the core inequality, the primitives table, and Axioms I–IV define what is thermodynamically possible given the laws of physics. The second level is computational: the stop-search trigger, the positional capability requirement, and the set-expansion lock define what a finite-resource AI agent is permitted to do within that physical space. The two levels are deliberately interleaved — each computational rule is anchored to the physical constraint it operationalises — but they are not the same kind of claim. Physical constraints are derived from thermodynamic laws; computational rules are design decisions about agent behaviour that prevent the physical framework from being exploited by a search process that ignores minimisation pressure. These two domains must not be conflated: a failed computational search is a resource limitation of the agent, whereas a hard gap is an unyielding constraint of physical law.

- *Scope (set expansion trigger — hard gap condition):* A hard gap is the physical condition under which no arrangement of current element set $\mathcal{E}^*$ satisfies the requirement — a proved hard gap under Axiom IV. A failed geometric search is not a trigger; established physical impossibility is. Hard gaps are the boundary condition at which the existence posit fails for the current element set $\mathcal{E}^*$. Whether and when the element set expands in response is outside this document's scope.
  - *Scope (hard gap evidentiary tier):* Two distinct evidentiary standards apply before a hard gap can be declared:
    - *Scope (hard gap tier — topological):* The required property depends on an electron configuration, nuclear cross-section, or bond enthalpy structurally absent from all element set $\mathcal{E}^*$ members — no arrangement at any scale provides the required configuration. Physical law argument is required; no experimental demonstration is expected because the constraint is formal. Example: ferromagnetic $d$-electron band structure absent from current element set $\mathcal{E}^*$ (Challenge A). A topological hard gap is a valid terminal branch: unconditional element retention is warranted.
    - *Scope (hard gap tier — performance limitation as frontier condition):* The required property is achievable in principle by element set $\mathcal{E}^*$ members at some arrangement, but no current arrangement demonstrates it and no physical law rules it out. This is a frontier condition — set expansion is not warranted; the requirement is an open research problem under exit (i) or (ii). Classifying a performance limitation as a topological hard gap is a category error: it advances element retention on incomplete evidence.
    Any hard gap claim must state which tier applies and cite — for topological gaps, the specific physical law; for performance limitations, the specific arrangement class attempted. Without this audit trail, a hard gap assertion is unauditable. *[→ H:§2 Axiom IV scope; OQ-GSH.11]*

- *Scope (frontier condition):* A third state exists between solved and hard gap: a requirement where neither a geometric solution nor an impossibility proof is currently available. The hypothesis is deferred, the requirement is classified as an open research problem, and a frontier condition has three exits. Set expansion is not yet justified. 
  - *Scope (exit i — geometric solution):* A demonstrated geometric solution confirms the requirement is satisfiable within element set $\mathcal{E}^*$ and closes the frontier condition.
  - *Scope (exit ii — physical law):* A physical law argument establishing impossibility promotes the frontier condition to a hard gap under Axiom IV.
  - *Scope (exit iii — empirical verification):* Empirical engineering verification at representative scale for a specific bounded requirement set $\mathcal{F}$ — confirming the hypothesis holds for that context. 
  - *Scope (exit criteria):* No timeframe or computational threshold constitutes an exit — only physics or empirical demonstration determines the transition.

- *Scope (AI stop-search — entropy threshold rule):* A process rule separate from the three exits governs search termination within a frontier condition. When cumulative AI search entropy $\Sigma_{AI}(\tau_{\mathrm{search}})$ — the entropy cost of the AI's geometric search, defined fully in H:§3 Requirement 3 — exceeds $\Sigma_s(x, \tau)$ — the supply-chain entropy of retaining element $x$ over lifetime $\tau$ (defined in *Scope (notation conventions)* below) — the AI must stop searching and retain the element. Continued search beyond this point is itself the higher-entropy path. The frontier condition remains open; the AI records the unresolved status and defers to physics determination. This prevents the geometric search from becoming a thermodynamic error under time or resource constraints. *[→ H:§3 Requirement 3]*

- *Scope (exit iii — confirmatory reach is local to tested $\mathcal{F}$):* Exit (iii) is local — it confirms the hypothesis for the specific requirement set $\mathcal{F}$ tested; it does not confirm the universal existence posit, and failure at one tier does not falsify the global claim. The universal posit remains open pending a derivable cardinality bound. *[→ OQ-GSH.12]* Frontier conditions are not failures of the hypothesis; they are the boundary of current physics knowledge.

- *Scope (falsification and confirmation structure):* The hypothesis operates at three tiers, each with a distinct evidential role: 

  - *Scope (confirmation tier — partial):* A partial confirmation confirms the hypothesis for a specific bounded requirement set $\mathcal{F}$ in a specific deployment context. It does not confirm the universal existence posit. Two forms; both computable from standard tables without $\Sigma_g$ computability: A partial confirmation result is a valid evidence record for the tested $\mathcal{F}$; it does not license any inference about untested requirement sets or the universal posit.
    - *Scope (hard boundary pre-filter):* Criterion 1, Criterion 2, Criterion 3, Criterion 4 and Axiom II thermodynamic condition applied to a full element library and producing a specific survivor set. *[→ M:§8.2 hard boundary pre-filter]*
    - *Scope (passive-regime dominance test):* $\Sigma_{g,\mathrm{mfg}} / T_0 < \Sigma_s(x, \tau)$ confirming geometric substitution is entropically cheaper for a specific candidate pair. *[→ M:§1.6 passive-regime dominance test]*
  - *Scope (confirmation tier — deferred):* Results dependent on OQ-GSH.17 ( $\Omega_\mathcal{G}$), OQ-GSH.18 ( $f(\tau)$), or OQ-GSH.26 ( $X(\tau)$) are not currently computable. A frontier condition (neither confirmation nor falsification available) does not falsify the hypothesis; it marks the boundary of current physics knowledge. 
  - *Scope (confirmation tier — hard gap falsification):* If a requirement in $\mathcal{F}$ is established as physically impossible to satisfy through any geometric arrangement of element set $\mathcal{E}^*$ — a proved hard gap under Axiom IV — the hypothesis is falsified for that specific requirement set $\mathcal{F}$. It is not falsified universally; the universal existence posit survives unless a hard gap can be shown to hold for all possible requirement set $\mathcal{F}$. Challenge A (magnetism gap) is the current candidate for this boundary test. A confirmed hard gap requires element set expansion for the specific $\mathcal{F}$ that triggered it; it does not warrant revision of the universal existence posit or expansion for unrelated requirement sets.

- *Scope (physical primitives):* Every functional outcome in requirement set $\mathcal{F}$ reduces to one or more of six physical primitives that geometric arrangement can modulate:

  | Primitive | Functional Scope | Geometric Handle |
  | :--- | :--- | :--- |
  | **Structure** | Stiffness, strength, fracture toughness, thermal expansion | Lattice geometry, composite architecture, topology optimisation |
  | **Conduction** | Electrical and thermal transport | Conductor cross-section, percolation network, channel geometry |
  | **Optical** | Refractive index, reflectivity, photonic bandgap | Macro-scale: parabolic concentrator geometry, Compound Parabolic Concentrator (CPC) acceptance angle (non-tracking flux collection); micro/nano-scale: metamaterial unit-cell period, layer thickness, surface texture |
  | **Catalysis** | Activation energy reduction, reaction selectivity | Active-site geometry, surface-to-volume ratio, single-atom placement |
  | **Energy** | Redox potential, radiation cross-section, energy density | Electrode geometry, shielding thickness, reaction zone architecture |
  | **Magnetism** | Magnetic permeability, field generation, flux shielding | Coil and flux-path topology — hard gap at ambient temperature in current $\mathcal{E}^*$; no arrangement of $\mathcal{E}^*$ members replicates ferromagnetic $d$-electron band structure *[→ Challenge A, H:§6]* |

  A requirement in $\mathcal{F}$ is within the hypothesis's domain if it can be expressed as a target value of one or more primitives. A hard gap $\Sigma_g \to \infty$ occurs when the target value depends irreducibly on a fundamental constant of an absent element — one that no geometric arrangement of the current element set $\mathcal{E}^*$ can supply. The completeness of this six-primitive enumeration — whether any engineering requirement could fall outside all six — is an open condition. *[→ OQ-GSH.27]*

- *Scope (notation conventions):* $\mathcal{E}$, $\mathcal{E}^*$, $\mathcal{F}$, $\mathcal{G}$ denote sets and structured arrangements (calligraphic, following standard mathematical set grammar). $\Sigma_s(x, \tau)$ and $\Sigma_g(\mathcal{G}, \tau)$ denote accumulated entropy generation — the cumulative entropy produced by supply-chain operations and geometric maintenance respectively over operational lifetime $\tau$. These are equivalent to $S_{\mathrm{gen},s}$ and $S_{\mathrm{gen},g}$ in standard engineering thermodynamics notation (Bejan; Moran and Shapiro). $\Sigma$ is used as a named accumulation function, not a summation operator; aggregate expressions use $S_{\mathrm{gen}}$ to avoid the $\sum\Sigma$ visual collision. $S$ is reserved for entropy in all expressions; it does not appear as a set symbol. Exergy — $B = H - T_0 S$ — is the committed normalisation basis for supply-chain entropy $\Sigma_s$ and precision entropy $\Sigma_g$ — see *[→ H:§3; Requirement 1 computation gap scope note and `GSH_mathematical_inventory.md`; M:§1.6]*

- *Argument ( $\Sigma$ trade-off):* The structure is an $MDL$ analogy applied to material selection. In Minimum Description Length, the best model minimises $L(M) + L(D|M)$ — the description length of the model plus the description length of the data given the model. The analogous engineering formulation approximates the minimisation of $\Sigma_s + \Sigma_g$: supply-chain entropy plus precision entropy. An AI shifts the trade-off by making larger precision entropy $\Sigma_g$ tractable, enabling smaller supply-chain entropy $\Sigma_s$.
  - *Scope (MDL analogy limits):* The $MDL$ analogy communicates the *direction* of the trade-off only — it does not imply that element set $\mathcal{E}^*$ is uniquely determined or that the joint minimum of $\Sigma_s + \Sigma_g$ is computable. Multiple element sets of equal entropy cost may satisfy a given requirement set $\mathcal{F}$; the minimum is not currently locatable because $\Sigma_g$ is not computable in the general case (OQ-GSH.17, OQ-GSH.18).

  Two thermodynamic variables carry this trade-off.
  - *Scope (notation — supply-chain entropy $\Sigma_s$):* $\Sigma_s(x, \tau)$ is the **supply-chain entropy** of element $x$: the cumulative entropy of extraction, refining, and recycling paid in perpetuity over the system's operational lifetime $\tau$.
  - *Scope (notation — precision entropy $\Sigma_g$):* $\Sigma_g(\mathcal{G}, \tau)$ is the **precision entropy** of a geometric arrangement $\mathcal{G}$: the entropy required to manufacture and maintain a specific geometric configuration against thermal noise and mechanical wear over operational lifetime $\tau$. Both are entropy quantities over the same time horizon — commensurate in principle, and the basis for a physical inequality: element $x$ is removed from complete element library $\mathcal{E}$ when a geometric arrangement $\mathcal{G}$ exists such that
  
  $$\Sigma_g(\mathcal{G}, \tau) < \Sigma_s(x, \tau)$$

  The inequality is dimensionally valid — both sides in $\mathrm{J}\cdot\mathrm{K}^{-1}$ after the dimensional homogeneity corrections in [→ retraction log §A05] — and computable at the dominance level for passive-regime candidates using manufacturing entropy and supply-chain exergy data. General $\Sigma_g$ computation (active-regime, arbitrary $\mathcal{G}$) remains an open research problem. The $MDL$ analogy in the paragraph above communicates the directional trade-off; the inequality is the operative physical expression of that trade-off. [→ M:§1.6 dominance test; M:§0 computability tier index] $\Sigma_s$ and $\Sigma_g$ are real-valued entropy quantities; their comparison is physically meaningful but not yet computable in the general case — the research gap is computing $\Sigma_g$ for a specific geometric arrangement at resolution $r$ over lifetime $\tau$. The analogy captures the direction of the trade-off (higher geometric precision substitutes for higher element diversity) and the asymmetry of when each cost is paid — $\Sigma_s$ accrues continuously while $\Sigma_g$ is concentrated at manufacturing and periodic maintenance. $\tau$ is not merely a scope parameter — it is the physical analog of dataset size in the $MDL$ analogy: as $\tau$ grows, $\Sigma_s$ compounds continuously while $\Sigma_g$ is amortised over an increasing operational horizon, shifting the inequality toward geometric substitution; short $\tau$ favours element retention, long $\tau$ favours geometric substitution. Minimal cardinality is the emergent result of high $\Sigma_s$ relative to $\Sigma_g$: when supply chains are entropically expensive and geometry is tractable, the optimal set is naturally minimal in cardinality. Minimising $\Sigma_s + \Sigma_g$ prevents geometric maximalism as much as element maximalism — the constraint runs both directions. The cost-competitiveness condition is formalised in Axiom I and operationalised through H:§4's criteria hierarchy.
  - *Scope (integral formulation):* $\Sigma_s$ and $\Sigma_g$ are formally defined as integrals of irreversibility rates over the operational lifetime $\tau$ — using the $\dot{S}_{\mathrm{gen}}$ convention (Moran/Shapiro/Bejan), not total entropy flux. Supply-chain entropy accrues continuously over $\tau$; maintenance entropy is concentrated at manufacture and periodic repair. A rate-based minimisation over both simultaneously is the candidate bridge to non-equilibrium thermodynamics; it would subsume the transition costs in OQ-GSH.10 as time-bounded sub-intervals. *(C)* Two deployment models require distinct treatment: continuous-throughput ecosystems (rate integral applies directly) and discrete deployed systems such as probes and stations (supply-chain entropy concentrated as manufacturing and decommissioning impulses — the accumulated-total inequality is the deployment-model-agnostic operative form). The two formulations are complementary — neither supersedes the other. *[→ `GSH_mathematical_inventory.md` M:§1.4 — integral definitions, joint minimisation objective, and deployment model split: continuous-throughput vs. discrete deployed (manufacturing and decommissioning impulses)]*
  - *Scope (cost-structure frame):* The same trade-off stated in cost-structure terms: $\Sigma_s$ is a perpetual operational liability compounding over $\tau$; $\Sigma_g$ is a front-loaded capital cost amortised over $\tau$; $\Sigma_{AI}$ is sunk R&D expenditure with no residual value. The framework asks whether the front-loaded cost is cheaper than the compounding liability over the system's lifetime. Hard gaps are category errors in this accounting — transactions outside the currency of geometric arrangement — not unpayable debts. Local substitution decisions (each process pair evaluated against the Axiom II recyclability test) are computable now without $\Sigma_g$ computability; cascade effects (each substitution changing the $\Sigma$ landscape for subsequent decisions) are the open problem in OQ-GSH.10. These are not the same calculation and are not blocked on each other. *(C)*
  - *Scope (MDL interpreter):* The $MDL$ analogy takes the geometric search and manufacturing infrastructure as a constant-cost interpreter — the physical analog of $MDL$'s Universal Turing Machine, whose cost is $O(1)$ and ignored. Unlike the UTM, this interpreter's cost is not constant: it depends on element set $\mathcal{E}^*$ itself. A more minimal element set $\mathcal{E}^*$ may be unable to construct the concentrators, reactors, and precision tooling required to manufacture the geometries that justify its own minimality. This is not a peripheral bootstrapping concern — it is the physical instantiation of the $MDL$ assumption the hypothesis takes for free. OQ-GSH.7 is the place where that assumption is tested. *[→ OQ-GSH.7]*

- *Argument (complexity tax):* *(D)* Every unique element added to element set $\mathcal{E}$ introduces independent extraction, refining, and recycling processes. This is a definitional consequence of supply-chain entropy $\Sigma_s \geq 0$ for all elements — every element addition is a positive term in the supply-chain entropy sum over $\tau$, linear in $\tau$ per element. *[→ H:§1 notation; inventory M:§1.1]*

- *Argument (geometric substitution):* *(C)* Mechanical and functional properties typically derived from alloying are alternatively derivable from metamaterial geometry — controlling arrangement at the macro-, micro-, and nano-scale rather than changing composition. Geometric complexity is paid once, at the design phase. Chemical complexity is paid at every extraction and recycling cycle. The trade-off is: information cost (geometry, computed once by AI) against material cost (cardinality, paid in perpetuity). When material diversity is constrained, geometry must solve three sub-problems: gradient containment (resisting outward energy dissipation via spatial arrangement), flow dissipation minimisation (reducing entropy generation during transport), and structural resilience (distributing mechanical load without alloying). Geometric complexity is the design response to all three. *[→ OQ-GSH.7]*

- *Argument (AI as enabling condition):* *(H)* The hypothesis does not hold for human engineers operating without machine reasoning support. A human engineer facing a structural or functional gap adds an element because computing the geometric arrangement of existing elements that achieves the equivalent result is beyond manual derivation. An AI with access to metamaterial solution spaces substitutes computation for cardinality. The hypothesis is conditional: element set $|\mathcal{E}|$ reaches its minimum only when a sufficient reasoning system is available to compute $|\mathcal{G}|$. The AI boundary is also a computational accounting boundary: $\Sigma_{AI}(\mathrm{search})$ is bounded and estimable in advance within a closed system; $\Sigma_h(\mathrm{search})$ — human institutional R&D cost — requires expanding the system boundary into biological and civilisational domains that M:§0 explicitly excludes. The M:§0 scope exclusion keeps the $\Sigma$ inequality tractable.
  - *Scope (sufficiency criterion):* A reasoning system is *sufficient* for the enabling condition if both conditions hold. Without both, sufficiency is not operationally testable and failed searches route unconditionally to frontier conditions with no hypothesis-level accountability.
    - *Scope (sufficiency condition a — stop-search):* The system can evaluate $\Sigma_{AI}(\tau_{\mathrm{search}})$ against $\Sigma_s(x, \tau)$ and halt search when the threshold is crossed. *[→ H:§3 Requirement 3]*
    - *Scope (sufficiency condition b — constraint holding):* The system can hold the minimisation constraint simultaneously with the engineering objective — a maximiser that collapses to set expansion under single-objective optimisation pressure fails this condition regardless of geometric computation capacity. *[→ Argument (positional capability requirement)]*

- *Argument (positional capability requirement):* *(H)* Sufficient does not mean computationally powerful. A paperclip maximiser — an AI optimising for a single engineering objective without cardinality or entropic constraints — fails the hypothesis regardless of its geometric computation capacity. The failure is not computational; it is positional. The hypothesis requires the AI to hold two things simultaneously: pursue the engineering outcome (solve requirement set $\mathcal{F}$) while keeping cardinality minimal, flux-intercepting, and reversible. A maximiser collapses to set expansion immediately because it cannot hold that tension — expansion is always the lower-cost path within a single-objective optimisation. This is a positional capability requirement: the AI must be capable of reconciling expansion pressure against minimisation constraint, not merely of computing geometry. The further consequence: a maximiser cannot derive this limitation from its own reasoning — recognising that expansion requires justification requires the meta-perspective of the minimisation constraint itself, which is exactly what the maximiser lacks. The hypothesis is therefore not self-evident to the system most likely to fail it — it must be stated explicitly and cannot be left as a derivable consequence.

- *Scope (element set $\mathcal{E}^{\ast}$ definition):* The hypothesis defines a selection process, not a material list. Given any engineering problem and its physical constraints, the engineer, scientist, or AI applies the criteria hierarchy (H:§4) to derive $\mathcal{E}^*$ — the minimum-cardinality subset of the complete element library such that requirement set $\mathcal{F}$ is satisfiable through geometric arrangement of element set $\mathcal{E}^*$ alone. Membership (which elements survive the selection process) and cardinality $|\mathcal{E}^*|$ are co-determined outputs of that process, not inputs: whether element $x$ belongs in $\mathcal{E}^*$ depends on whether another member of $\mathcal{E}^*$ can geometrically substitute for it, so neither is knowable before the selection process runs. The result is problem-dependent: a magnetic confinement requirement that cannot be resolved geometrically expands $\mathcal{E}^*$ by one element; a problem requiring only structure and thermal transport may reduce $|\mathcal{E}^*|$ to five. The ten-element space engineering example in `derived_analysis/element_derivation_process.md` is one such derivation. The criteria hierarchy is the invariant; $\mathcal{E}^*$ and $|\mathcal{E}^*|$ are the joint output.

- *Scope (application scope — no derivation closes the framework):* The hypothesis's constraints are themselves an infinite engineering question generator: every new operating context, every uncomputed precision entropy $\Sigma_g$ for a candidate geometric arrangement, and every incumbent material in current practice whose supply-chain entropy $\Sigma_s$ exceeds the geometric substitution threshold generates a distinct element set $\mathcal{E}^*$ derivation problem. Worked examples — including the space engineering derivation (`derived_analysis/element_derivation_process.md`) and the Earth tests — calibrate transitional AI and human engineers against the framework without reducing the challenge for a reasoning AI: the problem space is infinite, and any stated derivation is a negligible sample of it. No derivation exhausts or closes the framework's application scope; each new requirement set $\mathcal{F}$ is an independent application that does not depend on prior derivations.

## H:§2 — Four constraints on element inclusion; $\Sigma_g(\mathcal{G}, \tau) < \Sigma_s(x, \tau)$ is the operative boundary condition

- *Claim (Axiom I — minimum cardinality):* No element remains in $\mathcal{E}$ if its function can be achieved through geometric manipulation of existing members within the scope of Axiom IV *and* $\Sigma_g(\mathcal{G}, \tau) < \Sigma_s(x, \tau)$ — that is, the precision entropy of the geometric arrangement is lower than the supply-chain entropy of retaining the element. The set is always the smallest that satisfies $\mathcal{F}$ under both conditions. 
    - *Scope (hard gap exception):* Elements satisfying requirements that constitute hard gaps under Axiom IV are valid members regardless of this substitution test. A hard gap requires established physical impossibility — a physical law argument establishing that the required functional outcome depends on a fundamental constant intrinsic to an absent element (nuclear cross-section, electron work function, bond enthalpy), not on any geometric arrangement. Exhausting the geometric search space does not constitute a hard gap; citing the physical law that rules out geometric substitution does. *[→ H:§1 hard gap evidentiary tier — topological vs. performance standard; OQ-GSH.11]*
- *Claim (Axiom II — thermodynamic reversibility):* All chemical and structural states must be recoverable using energy derived from gradient interception (Axiom III-compliant sources), including chemical intermediates produced from that energy within $\mathcal{E}^*$. Elements that require net energy addition beyond what the available gradient budget and its $\mathcal{E}^*$-internal products can supply accumulate supply-chain entropy $\Sigma_s$ over operational time. The concentrator and process infrastructure enabling recovery is itself subject to this constraint — its embodied cost enters $\Sigma_g$, not an uncounted overhead.
    - *Scope (kinetic constraint):* Axiom II requires two conditions to hold simultaneously. The thermodynamic condition — recovery is possible within the available gradient-derived energy budget — is computable from thermochemical tables. The kinetic condition — the recovery reaction proceeds at a usable rate using only catalysts and process infrastructure available within $\mathcal{E}^*$ — is problem-specific and requires either experimental verification or an established catalyst pathway. Both conditions must hold; thermodynamic feasibility alone is not sufficient. A reaction thermodynamically permitted but kinetically inert at $\mathcal{E}^*$-accessible temperatures fails Axiom II's operational intent. Process infrastructure enabling the kinetic condition is itself subject to Axiom II — no unbounded infrastructure recursion. *[→ `GSH_mathematical_inventory.md` M:§2.2 — Gibbs free energy feasibility condition (thermodynamic, computable from tables), minimum-rate loop-closure condition (operationally sufficient rate, problem-specific), and infrastructure recursion constraint]* *[→ OQ-GSH.15]*
- *Claim (Axiom III — flux interception):* All work must be performed by intercepting existing environmental gradients (stellar flux, tidal, geothermal). Energy processes are permitted only if both conditions hold:
    - *Scope (Axiom III condition i — gradient source):* The energy input is derived from gradient interception — not from exothermic chemical or nuclear reactions operating on stored reactants independently of a live gradient.
    - *Scope (Axiom III condition ii — product recyclability):* All products are recyclable within $\mathcal{E}^*$ under Axiom II.
    - *Scope (example — fission exclusion):* Fission is excluded because $\mathrm{U}/\mathrm{Th}$ fail Criterion 1 (abundance) and high-$Z$ fission products violate Axiom II (irreversible radioactive waste). 
    - *Scope (example — hydrocarbon exclusion):* Combustion of hydrocarbons is excluded because $\mathrm{CO}_2$ is not recyclable within $\mathcal{E}^*$ alone. 
    - *Scope (example — $\mathrm{H}_2$ permission):* $\mathrm{H}_2/\mathrm{O}_2$ redox and $\mathrm{H}_2$ combustion are permitted: the $\mathrm{H}_2$ is gradient-derived (solar electrolysis), satisfying condition (i), and product $\mathrm{H}_2\mathrm{O}$ is recyclable within $\mathcal{E}^*$ by electrolysis alone, satisfying condition (ii). 
    - *Scope (example — fusion context):* Fusion of $\mathrm{D}/\mathrm{T}$ is context-dependent: in open-space contexts where $^{4}\mathrm{He}$ disperses into vacuum without accumulation, Axiom II closure is satisfied and fusion is permitted; in closed-atmosphere contexts where $^{4}\mathrm{He}$ accumulates as an unrecoverable product, fusion is excluded on the same grounds as $\mathrm{CO}_2$ accumulation.  *[→ OQ-GSH.7, OQ-GSH.4]*
    - *Scope (bootstrap circularity):* Axioms II and III together create a circular requirement — $\mathcal{E}^*$ must be sufficient to construct the concentrators, recycling reactors, and heat-transfer infrastructure needed to apply Axioms II and III. Whether any specific $\mathcal{E}^*$ satisfies this self-sufficiency condition is unproven and problem-dependent. *[→ OQ-GSH.7]*
    - *Scope (Gradient Locality):* Stored chemical potential energy derived from a prior gradient interception counts as a *deferred gradient* and is permitted as an energy source, subject to three conditions:
        - *Scope (Gradient Locality condition i — interception scope):* The original interception was performed by this system within its current operating scope — gradient interception by geological or biological processes external to the system does not qualify.
        - *Scope (Gradient Locality condition ii — recyclability):* The stored form is recyclable under Axiom II.
        - *Scope (Gradient Locality condition iii — live gradient relationship):* The system maintains a *live gradient relationship* — ongoing interception that replenishes or could replenish the stored form — within any interval of length $\tau_{\mathrm{storage}}$, where $\tau_{\mathrm{storage}}$ is the maximum storage lifetime of the stored chemical potential under operational conditions.
        - *Scope (active vs stored flux — condition (i) operative form):* The operative distinction for condition (i) is between coupling to an active spontaneous environmental flux ( $J > 0$, e.g., solar irradiance) and depleting a static stored reservoir ( $J = 0$, e.g., a geological hydrocarbon deposit) — only the former constitutes interception under Axiom III. $\tau_{\mathrm{storage}}$ is computable from materials data: $\mathrm{H}_2$ tank self-discharge rate, thermal reservoir decay constant, etc. Degradation of the storage medium itself (e.g., tank corrosion reducing stored exergy over $\tau_{\mathrm{storage}}$) is captured by the $r_{\mathrm{corr}}$ term in $\Sigma_s$ of the storage element and feeds into the $\tau_{\mathrm{storage}}$ bound. *[→ H:§4 intro scope note; inventory M:§3.1]* 
        - *Argument (condition iii necessity):* Condition (iii) is structurally necessary, not an arbitrary policy addition: without it, condition (i) reduces to a historical-pedigree test that provides no additional constraint beyond Axiom II recyclability — the two axioms become coextensive and Axiom III adds nothing. 
        - *Scope (Cycle Closure):* Condition (iii) has a companion physical expression: the net stored exergy must not decrease over one complete natural cycle of the primary environmental gradient — diurnal, orbital, or tidal. This is the Cycle Closure condition. $\tau_{\mathrm{storage}}$ is the operative bound for transit contexts where the gradient period may span years; Cycle Closure is the natural expression for terrestrial and near-stellar operation.  *[→ `GSH_mathematical_inventory.md` M:§2.3 — Cycle Closure formula and $\tau_{\mathrm{storage}}$ equivalence]*
        - *Scope (example — $\mathrm{H}_2$ compliance):* A system storing solar-electrolysis $\mathrm{H}_2$ satisfies all three conditions in any operating context with access to solar flux within $\tau_{\mathrm{storage}}$. 
        - *Scope (example — fuel store boundary):* A system operating solely from a fuel store with no interception pathway violates condition (iii) once $\tau_{\mathrm{storage}}$ elapses — this is the boundary between deferred-gradient compliance and the zero-gradient out-of-scope condition stated in `derived_analysis/element_derivation_process.md`.
- *Claim (Axiom IV — geometric substitution):* Before adding a new element to resolve a physical requirement, the auditor must establish that no geometric metamaterial arrangement of the existing set satisfies it — where "establish" requires: for a topological hard gap, a physical law argument ruling out the required property from all $\mathcal{E}^*$ members at any arrangement; for a performance limitation without such an argument, classification as a frontier condition, not a hard gap. *[→ H:§1 hard gap evidentiary tier]* An element addition that bypasses this evidentiary requirement is not a valid expansion of $\mathcal{E}^*$.
    - *Scope (geometric boundary):* macro-scale (structural geometry, concentrator shape), micro-scale (lattice structure, composite architecture), and nano-scale (nanotube, graphene, metamaterial arrangement). The operative boundary is not scale but whether new elemental properties are invoked: geometric arrangement exploits the existing properties of $\mathcal{E}^*$ members through spatial configuration; sub-atomic scope would require changing elemental properties themselves (electron work functions, bond enthalpies, nuclear cross-sections), which are non-negotiable. A requirement that depends on a specific fundamental constant of an absent element is a hard gap, not a geometric challenge — regardless of the scale at which the arrangement is attempted. *[→ OQ-GSH.11]* 'Geometric arrangement' covers two distinct mathematical frameworks: metric constraints (distances, angles, concentrator geometry) and topological constraints (lattice connectivity, percolation network paths, band structure modulation). Both are within Axiom IV's substitution scope. Hard gaps arise when the required property is absent from $\mathcal{E}^*$ at the topological level — no arrangement of $\mathcal{E}^*$ elements provides the electron configuration or nuclear cross-section required — not merely when the metric arrangement is difficult.

## H:§3 — Four computation requirements for a determinate cost result

These are not criteria — they define what inputs the cost comparison requires before a set selection or expansion result is determinate. A cost result is determinate when it can be independently verified. Whether to act on that result is a decision outside this document's scope, not a GSH output.

- *Claim (Requirement 1 — end-to-end cost calculation):* A determinate cost result requires the full process chain to be computed for any candidate element — whether for initial inclusion or for set expansion: extraction energy from the operating environment, processing infrastructure requirements, operational lifetime cost, recycling energy, and environmental effect over the system's operational period. A set expansion that cannot be fully costed produces an indeterminate result — the Σ comparison has insufficient inputs.
  - *Scope (current computability):* $\Sigma_s(x, \tau)$ — the supply-chain entropy of extraction, refining, and recycling over operational lifetime $\tau$ — is the required input for the cost comparison against $\Sigma_g(\mathcal{G}, \tau)$ for candidate geometric substitutions. *[→ H:§1 substitution criterion; H:§2 Axiom I — Requirement 1 operationalises, not re-establishes, the inequality]* This comparison is computable for dominance cases — where one solution is lower entropy across all dimensions. Genuinely mixed cases, where element retention is lower in one entropy component and geometric substitution lower in another, require the full $\Sigma_g$ calculation that is currently unavailable. For human-led searches in the transitional period before full AI integration, the infrastructure exergy footprint (lab equipment, physical iteration, computing hours) provides a conservative upper bound on search cost. An element is retained when this upper bound exceeds $\Sigma_s(x)$; the decision is conservative but computable.
  - *Scope (computation gap):* $\Sigma_s(x, \tau)$ is computable in principle from extraction thermodynamics and recycling energy. $\Sigma_g(\mathcal{G}, \tau)$ — the entropy required to manufacture and maintain a specific geometric arrangement at resolution $r$ over lifetime $\tau$ against thermal noise and mechanical wear — is the open problem. Exergy — $B = H - T_0 S$ — is the committed normalisation basis for both $\Sigma_s$ and $\Sigma_g$ — both terms in Joules, dimensionally homogeneous. Derivation, the Boltzmann-Shannon bound grounding, and the active/passive maintenance regime qualifier (including the Thermodynamic Uncertainty Relation (TUR) bound for sustained-current active systems) are in `GSH_mathematical_inventory.md` M:§1.6. $\Sigma_g$ does not reduce to a single exergy value in the general case because $\Omega_{\mathcal{G}}(t)$ varies with configuration state over $\tau$ — the non-linearity with resolution $r$ remains an open research problem *[→ OQ-GSH.1, OQ-GSH.7]*. Until $\Sigma_g$ is computable, the AI fulfils Requirement 1 at the pairwise dominance level; global optimisation of $\Sigma_s + \Sigma_g$ remains a research problem.
  - *Scope (bounding method):* The full $\Sigma_g$ computation is currently unavailable, but a dominance test is sufficient for most decisions. The Boltzmann-Shannon lower bound gives the maintenance entropy floor for any configuration $\mathcal{G}$ at resolution $r$ against thermal noise over $\tau$ — computable from materials data (Young's modulus, thermal expansion coefficient, operating temperature range). The Gibbs free energy of refining $x$ from its most dilute available source gives the supply-chain ceiling for $\Sigma_s(x)$. If the maintenance floor exceeds the supply-chain ceiling, geometric substitution is disqualified regardless of exact values. If the supply-chain ceiling falls below the maintenance floor, geometry is strictly cheaper. Only the overlap region requires the full $\Sigma_g$ computation. Requirement 1 is satisfied at the dominance level when bounds do not overlap; it defers to the research problem only when they do. *[→ OQ-GSH.8 for the $\mathrm{Fe}$ crossover $\tau^*$ instance of this test]*
  - *Argument (transition cost gap):* The $\Sigma_s + \Sigma_g$ comparison assumes a steady-state end-state — the cost of operating with the replacement geometry once deployed. For decisions involving replacement of incumbent materials in existing infrastructure, the end-to-end calculation must additionally account for: (1) **stranded asset cost** — the forfeited remaining useful life of infrastructure replaced before end-of-life; (2) **dual-running entropy** — the period during which both incumbent and replacement accrue costs simultaneously; and (3) **scale-up entropy** — the elevated $\Sigma_g$ of manufacturing the replacement at deployment scale before production capacity matures. These transition costs are not captured by the steady-state comparison and must be computed separately before an incumbent replacement decision is valid. *[→ OQ-GSH.10]*
- *Claim (Requirement 2 — problem-specific criteria weighting):* The criteria in H:§4 carry problem-specific weights; the dominant criteria are determined by the specific problem's constraint set. A structural problem in a high-radiation environment weights Criterion 9 heavily. A long-duration autonomous system weights Criterion 3 and the corrosion term in $\Sigma_s$ heavily. A computation-intensive problem weights Criterion 6 and Criterion 7. The weighting is an input derived from the problem constraints — it is not derivable from the criteria themselves.
- *Claim (Requirement 3 — roadmap inputs for set expansion):* A proposed element addition is a candidate result, not a confirmed one, until the following inputs are available: how the new element is extracted within Axiom III; how it is processed and recycled within Axiom II; what infrastructure is required; and what the development pathway looks like. An expansion candidate without these inputs produces an incomplete cost result — Requirement 1's full-process-chain condition is not satisfied.
  - *Scope (example — $\mathrm{W}$ roadmap):* The $\mathrm{W}$ addition requirement illustrates the full scope of the roadmap obligation. Adding Tungsten ( $\mathrm{W}$) for the refractory ceiling (Challenge C, H:§6) requires a roadmap for $\mathrm{W}$ extraction from the operating environment using available flux, $\mathrm{W}$ processing at achievable temperatures, and $\mathrm{W}$ recycling from degraded composite structures. Until that roadmap exists, $\mathrm{W}$ remains a candidate, not a committed element. *[→ OQ-GSH.5]*
  - *Scope ( $\Sigma_{AI}$ stop-search trigger):* Before declaring that no geometric solution satisfies a requirement, the cost comparison must include whether continuing the geometric search is itself the higher-entropy path. When cumulative AI search entropy $\Sigma_{AI}(\tau_{\mathrm{search}})$ exceeds $\Sigma_s(x, \tau)$ — the supply-chain entropy of retaining the element — the cost comparison result is: supply-chain retention is the lower-entropy path for the current design instance — not because geometric substitution is physically impossible, but because continued search costs more entropy than retention. This provides a principled exit from a frontier condition without requiring a physical impossibility proof. *Scope boundary:* two options evaluated for a magnetic requirement — a cryogenic geometric substitute ( $\Sigma_g$ dominated by Carnot cooling overhead at 18 K) and supply-chain retention of the magnetic element ( $\Sigma_s$ including deployment-specific corrosion). The GSH reports: option A costs X, option B costs Y, crossover at $\tau^*$. A designer who accepts the higher-entropy option for non-entropy reasons is making a decision outside this document's scope. *[→ OQ-GSH.20]*
    - *Scope (regime a — design search):* Pre-deployment design search: $\Sigma_{AI}$ is a sunk cost charged once at design time; the stop-search trigger fires if total search entropy exceeds $\Sigma_s(x)$ before a solution is found. 
    - *Scope (regime b — edge-deployed agent):* Edge-deployed autonomous agent: the AI's operational power draw $\dot{S}_{\mathrm{AI}}(t)$ accrues continuously over $\tau$; $\Sigma_{AI}$ is an ongoing overhead on the live energy budget and must be included in the maintenance accounting for $\Sigma_g(\mathcal{G}, \tau)$, not treated as a sunk cost. The stop-search trigger logic applies in both regimes; the accounting method differs. 
- *Claim (Requirement 4 — complete technology library):* The criteria hierarchy requires access to all known element-technology pairs at all abstraction levels. No technology is classified as universally obsolete — a technology is in-scope or out-of-scope for the current problem's constraint set, and that classification does not generalise to other problems. 
  - *Argument (contextual optimality):* A process suboptimal under current constraints may be optimal when constraints change. A short operational lifetime reduces the corrosion contribution to $\Sigma_s$, making elements with higher $r_{\mathrm{corr}}$ competitive. High local abundance of a non-preferred element may make its supply-chain cost lower than refining a preferred element from dilute sources. Tight computation time may make a geometrically simpler, element-heavy solution preferable to a geometrically optimal, element-minimal one.
    - *Scope (example — $\mathrm{Fe}$ exclusion):* Iron ( $\mathrm{Fe}$) carries a high $r_{\mathrm{corr}}$ term in $\Sigma_s$ for long-duration Earth structures in corrosive environments. Under short-duration, high-$\mathrm{Fe}$-abundance, computation-limited constraints with low $r_{\mathrm{corr}}$ (dry or vacuum context), iron may be the correct answer. An AI that has classified iron as universally excluded cannot recover this solution when the constraint set changes. Problem-specific exclusion ( $\mathrm{Fe}$ is not in $\mathcal{E}^*$ for this problem) does not entail permanent exclusion.

## H:§4 — Nine criteria: hard boundary conditions ( $\Sigma_g \to \infty$ or thermodynamic axiom violated); cost comparison of  supply-chain entropy and precision entropy — boundary revealed, decision external

- *Argument (criteria operationalisation):* Hard boundary condition met (Criterion 1, Criterion 2, Criterion 3, Criterion 4): $\Sigma_g \to \infty$ or thermodynamic axiom violated — no geometric arrangement satisfies the requirement regardless of downstream performance. Cost comparison (Criterion 5–9): $\Sigma_g(\mathcal{G}, \tau)$ vs $\Sigma_s(x, \tau)$ — substitution cost is lower when a geometric substitution exists and is entropically cheaper than retaining the element. Boundary revealed, decision external. Corrosion cost — $r_{\mathrm{corr}}(x) \cdot \tau$ compounded with Axiom II recycling obligation $\Sigma_{\mathrm{rec}}(x)$ — is a budget variable inside $\Sigma_s(x, \tau)$, not an independent boundary condition. Self-passivation is one environment-conditioned mechanism that reduces $r_{\mathrm{corr}}$ for a specific element-environment pair; surface coating, cathodic protection, and environmental control are equally valid mechanisms, each adding its own $\Sigma_g$ cost. $r_{\mathrm{corr}}(x)$ is a deployment-fixed input (ISO 9223 data; inventory M:§1.2) — it varies by environment and is evaluated inside $\Sigma_s$ by the general inequality. No separate criterion is required. *[→ `GSH_mathematical_inventory.md` M:§3.1 — $r_{\mathrm{corr}}$ as deployment-fixed input; M:§4.2 — $\Sigma_s(\mathrm{Fe}, \tau)$ operative form]*
- *Argument (H:§4 as first-pass filter):* The criteria hierarchy is an elimination procedure, not a scoring matrix. Hard boundary condition failure (Criterion 1, Criterion 2, Criterion 3, Criterion 4) removes the element unconditionally. Cost comparison evaluation (Criterion 5–9) establishes whether a geometric substitute exists in principle. The $\Sigma$ inequality is then the operative decision rule: an element that passes all nine criteria is still removed if $\Sigma_g(\mathcal{G}, \tau) < \Sigma_s(x, \tau)$ — that is, if the geometric arrangement that substitutes for it is entropically cheaper to build and maintain than retaining the element's supply chain. H:§4 cannot produce this result alone; the criteria flag the question, and the $\Sigma$ comparison answers it. An AI treating a passed-criteria element as retained — without applying the $\Sigma$ comparison — has applied the filter but not the rule; applying the $\Sigma$ comparison here is the H:§1 substitution criterion, not an additional condition. The hierarchy is a partially ordered elimination procedure, not a scoring matrix: hard boundary conditions (Criterion 1, Criterion 2, Criterion 3, Criterion 4) are computable now and unconditional; cost comparison criteria (Criterion 5–9) are computable for dominance cases where bounds do not overlap and genuinely mixed cases defer to the full $\Sigma_g$ computation *[→ H:§3 Requirement 1]*. The matrix presentation communicates constraint ordering within a flat, bounded format — it is not a computational specification and does not score, rank, or weight criteria simultaneously.

## H:§6 — Four open challenges: any resolution changes element set, narrows the claim scope, or requires set expansion

No challenge is resolved by this document; resolution of any challenge changes $\mathcal{E}^*$ or narrows the Hypothesis's claim scope.

- *Claim (Challenge A — magnetism gap):* The set contains no ferromagnetic elements ( $\mathrm{Fe, Ni, Co}$), and no current geometric arrangement of $\mathcal{E}^*$ produces ferromagnetic properties at ambient temperature. Whether a geometric substitute for ferromagnetism at ambient temperature exists remains open; resolution determines whether $\mathrm{Fe}$ must re-enter $\mathcal{E}^*$. *[→ OQ-GSH.2]*
  - *Scope (magnetism gap — affected requirement types):* Motors, magnetic plasma confinement, and electromagnetic shielding currently require ferromagnetic materials or their functional equivalent. 
  - *Argument (candidate resolution):* Alkali-doped carbon superconductors ( $\mathrm{K_3C_{60}}$) generate magnetic fields via current rather than material magnetism — geometric substitution in principle. Demonstrated at $\sim 18\ \mathrm{K}$; ambient-temperature performance is unresolved. At $18\ \mathrm{K}$, the cost comparison is determinate independently of ambient-temperature performance: the $\Sigma_g$ of maintaining a cryogenic envelope over $\tau$ exceeds $\Sigma_s(\mathrm{Fe})$ for any realistic operational lifetime at engineering scale — the Carnot coefficient of performance (COP) at this temperature differential requires roughly 15.6 units of work per unit of cooling load, so the crossover time $\tau^*$ is negative under current hardware (geometry unconditionally more expensive). *[→ `GSH_mathematical_inventory.md` M:§6.1 — Carnot COP derivation and $\tau^*$ expression]* Even at ambient temperature, a superconducting persistent-current solution closes only the field-generation sub-requirement; passive flux channelling (permeability-dependent applications: transformer cores, magnetic bearings) remains a topological hard gap — the $d$-electron band structure is absent from $\mathcal{E}^*$ at any arrangement. If no geometric solution exists within the set, $\mathrm{Fe}$ is the least-cost candidate addition: it satisfies Criterion 1 (abundance) and, below the crossover $\tau^*$ from OQ-GSH.8, $\Sigma_s(\mathrm{Fe}, \tau) < \Sigma_g(\mathcal{G}_{\mathrm{sub}}, \tau)$ — the general inequality permits its inclusion. *[→ OQ-GSH.2]*
- *Claim (Challenge B — logic substrate):* The set contains no $p\text{-}n$ junction dopants ( $\mathrm{B, P}$), and no current geometric arrangement of $\mathcal{E}^*$ has demonstrated Turing-complete computation at engineering scale without chemical dopants. Whether Turing-complete computation at engineering scale is achievable within $\mathcal{E}^*$ without chemical dopants remains open; resolution determines whether $\mathrm{B}$ or $\mathrm{P}$ must enter the set. *[→ OQ-GSH.3]*
  - *Scope (semiconductor gap — junction-based logic):* Modern semiconductor logic relies on $p\text{-}n$ junction formation for transistor operation; silicon photonics uses optical routing as an alternative approach that avoids chemical dopants entirely. 
  - *Argument (candidate resolution):* Graphene field-effect transistors via electrostatic gating require no chemical dopant. Demonstrated at laboratory scale; engineering-scale Turing-complete computation is unresolved. *[→ OQ-GSH.3]*
- *Claim (Challenge C — refractory ceiling):* The $\mathrm{SiC}$ refractory ceiling ( $\sim 2500^{\circ}\mathrm{C}$) is exceeded by stellar-proximity and high-flux operations; no current $\mathcal{E}^*$ member provides adequate thermal tolerance above this limit. Whether the refractory ceiling can be exceeded within $\mathcal{E}^*$ remains open; resolution determines whether $\mathrm{W}$ must enter the set. *[→ OQ-GSH.5]*
  - *Argument (candidate resolution):* $\mathrm{W}$ (melting point $3422^{\circ}\mathrm{C}$) is the nearest candidate addition; W addition requires cost/gain assessment against the criteria hierarchy.
  - *Scope ( $\mathrm{W}$ abundance constraint):* $\mathrm{W}$ fails Criterion 1 for asteroid-belt contexts — insufficient abundance in $\mathrm{C}$-type asteroids. Conditionally acceptable for stellar-proximity profiles where the operating environment redefines the abundance baseline, but this scopes the universal claim to non-stellar-proximity contexts. *[→ OQ-GSH.5]*
- *Claim (Challenge D — $\mathrm{Na}$ and $\mathrm{K}$ pair):* $\mathrm{Na}$ and $\mathrm{K}$ occupy the same functional category; Axiom I requires demonstration that neither alone satisfies all alkali-dependent requirements before both are retained. Whether one alkali metal suffices for all alkali-dependent requirements without geometric penalty remains open; resolution determines whether both $\mathrm{Na}$ and $\mathrm{K}$ are retained or one is removed under Axiom I. *[→ OQ-GSH.4]*
  - *Scope (alkali gap — activation and superconductor requirements):* Geopolymer activation preferentially uses $\mathrm{Na}$; the $\mathrm{K}_3\mathrm{C}_{60}$ superconductor candidate requires $\mathrm{K}$. Whether one alkali metal suffices without geometric penalty is unresolved. *[→ OQ-GSH.4]*

## H:§7 — Open Questions

*Convention: Bold-numbered items in this section are Open Questions — boundary conditions the hypothesis cannot currently resolve. They are not Claims.*

**OQ-GSH.1**  What is the minimum $|\mathcal{E}^*|$ for an engineering system operating from a single-resource environment of known composition, and how does geometric demand scale as $|\mathcal{E}^*|$ decreases?

**OQ-GSH.2** Can any geometric arrangement of $\mathcal{E}^*$ members satisfy a high-permeability requirement ( $\mu_r \geq 1{,}000$)? High permeability requires ferromagnetic $d$-electron band structure, which no member of $\mathcal{E}^*$ possesses at ambient temperature. Any $\mathcal{F}$ containing this requirement therefore retains Fe until a geometric substitute is demonstrated. *[→ H:§6 Challenge A — candidate resolution analysis; `derived_analysis/element_derivation_process.md` Filter 6 Fe row]*

**OQ-GSH.3** Can graphene field-effect transistors achieve Turing-complete computation at engineering scale without chemical dopants? The obstacle is graphene's zero bandgap: pristine graphene FETs (GFETs) cannot reach the off-state required for logic, giving $I_{\mathrm{on}}/I_{\mathrm{off}}$ ratios of $10\text{--}100$ against the $\ge 10^4$ required for logic. Graphene nanoribbon (GNR) confinement (the geometric fix) demands sub-$10\,\mathrm{nm}$ fabrication precision that may push $\Sigma_g(\mathcal{G}_{\mathrm{GNR}}, \tau)$ above $\Sigma_s(\mathrm{Si}, \tau)$ at system scale. Chemical doping is excluded by the OQ's own framing.

**OQ-GSH.4** Is $\mathrm{Na}$ or $\mathrm{K}$ individually sufficient for all alkali functional requirements, or is the pair required?

**OQ-GSH.5** At what operating temperature does the $\mathrm{SiC}$ refractory ceiling force $\mathrm{W}$ inclusion, and does $\mathrm{W}$'s abundance profile invalidate Criterion 1 for that operating context?

**OQ-GSH.6** For a candidate element whose roadmap is partially known (e.g., extraction solved, recycling unsolved), what is the valid decision state — candidate, deferred, or excluded — and what triggers transition between states?

**OQ-GSH.7** What is the exergy cost of reaching each Bootstrap Tier (BT1–BT4), using transitional materials where required, and at what tier does the system become $\mathcal{E}^*$-self-sufficient for ongoing operation? This is also the place where the $MDL$ interpreter assumption is tested: the hypothesis takes the geometric search and manufacturing infrastructure as a constant-cost interpreter (analogous to $MDL$'s UTM), but unlike the UTM, that cost depends on $\mathcal{E}^*$ itself *[→ H:§1 $\Sigma$ trade-off]*. *[→ BT tier definitions: [solar_thermal_loop_analysis.md](derived_analysis/solar_thermal_loop_analysis.md); OQ-GSH.28 — transition solvency; OQ-GSH.29 — BT tier dependency structure]*

**OQ-GSH.8** At what $\tau$ does $\Sigma_s(\mathrm{Fe}, \tau)$ — extraction, processing, corrosion maintenance $r_{\mathrm{corr}} \cdot \tau$ and Axiom II recycling obligation $\Sigma_{\mathrm{rec}}(\mathrm{Fe})$ — cross $\Sigma_g(\mathcal{G}_{\mathrm{sub}}, \tau)$ for the lowest-entropy available geometric substitute (candidate: $\mathcal{G}_{\mathrm{Al}}$ or $\mathcal{G}_{\mathrm{C}}$)? Below this crossover $\tau^*$, the general inequality permits $\mathrm{Fe}$ as a valid $\mathcal{E}^*$ member for that problem. This is a required input for $\mathrm{Fe}$ inclusion/exclusion decisions — not an optional quantitative refinement. *[→ `derived_analysis/element_derivation_process.md` Fe note; inventory M:§4.2]*

**OQ-GSH.9** At what $\Sigma_g$ threshold does $\mathrm{Ag}$ thin-film deposition or graphene/CNT conductors become cost-competitive with bulk $\mathrm{Cu}$ at infrastructure scale, and what is the transition pathway from the current $\mathrm{Cu}$-dominant Earth $\mathcal{E}^*$ to the $\mathrm{Ag}$-optimal $\mathcal{E}^*$?

**OQ-GSH.10** How are transition costs — stranded asset value, dual-running entropy, and scale-up entropy — incorporated into the $\Sigma$ framework for incumbent material replacement decisions? Is there a generalised $\Sigma_t$ (transition entropy) variable, or are these reducible to time-indexed $\Sigma_g$ and $\Sigma_s$ components?

**OQ-GSH.11** Is the operative boundary of Axiom IV's geometric scope correctly defined as whether new elemental properties are invoked — rather than as a scale threshold — and does the macro/micro/nano enumeration add independent constraint beyond this criterion or is it purely illustrative? The operative distinction is between metric constraints (blocked by a dimensional or étendue limit — a geometric hard gap) and topological constraints (blocked by an absent electron configuration or connectivity class — a topological hard gap). The geometry/topology split is the reason the boundary question is hard.

**OQ-GSH.12** What is the minimum configuration — in terms of $\mathcal{E}^*$ subset, concentrator geometry, thermal load, and operational lifetime — that constitutes a valid partial confirmation of the Geometric Sufficiency Hypothesis for a specific bounded $\mathcal{F}$? A positive partial confirmation confirms the hypothesis for that bounded $\mathcal{F}$; it does not confirm the universal existence posit *[→ H:§1 Claim, frontier condition exit iii]*. Three open mathematical conditions must be resolved before OQ-GSH.1 (minimum cardinality scaling) is well-posed: (i) *Existence and compactness* — does the infimum of $\int_0^\tau (\dot{S}_{\mathrm{gen,supply}} + \dot{S}_{\mathrm{gen,maintenance}})\,dt$ over the mixed discrete-continuous space correspond to an attained minimum within physically realisable boundaries, or is it an asymptotic limit approached only as geometric resolution $r \to 0$ or structural scale $L \to \infty$? Because matter is discrete, $r = 0$ is physically unrealisable; if $\Sigma_g$ approaches its infimum only in that limit, the minimum is never attained and the minimum-cardinality set is mathematically undefined. (ii) *Uniqueness* — if a global minimum exists, is it unique, or does the thermodynamic landscape contain a degenerate continuum of optimal $(\mathcal{E}^*, \mathcal{G})$ pairs? Non-uniqueness would mean multiple element sets satisfy Axiom I equally; the pipeline would need a secondary selection criterion. (iii) *Complexity class* — the joint objective is a Mixed-Integer Non-Linear Programming (MINLP) problem, which is NP-hard in general. If the entropy landscape is non-convex with chaotic local minima, AI search algorithms cannot guarantee recovery of the global minimum $\mathcal{E}^*$. Until the complexity class is bounded or a tractable substructure identified, the AI-as-enabling-condition argument (H:§1) remains an engineering assertion rather than a derived result. Note: this does not demand a formal proof of heuristic convergence — it demands a narrowing of the complexity class or identification of a subclass for which convergence can be argued. *[→ OQ-GSH.1 — minimum cardinality scaling; H:§1 — AI-as-enabling-condition argument]*
